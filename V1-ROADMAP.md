# Harness v1 Roadmap

## 목표

v1은 `실사용 개선` 버전이다.
핵심은 아래 3가지다.

1. 실제 task에 하네스를 적용했을 때 덜 번거롭다
2. 추론 품질을 해치지 않고 토큰 사용을 줄인다
3. 형식 검증과 반복 출력은 로직으로 밀어낸다

## 현재 반영 완료

- `sync/context` 구조화 요약
- `/plan` 전 계획서 lint
- `/work` diff 메타데이터 요약
- v0 문서 아카이브 구조 분리

## 다음 개선 우선순위

1. `task-hpa-manifest` 실사용 적용
2. PR 본문 정형 부분 자동화
3. state 요약 출력 추가
4. task별 참조 문서 목록화
5. 결과를 보고 게이트/출력 길이 재조정

## 완료 기준

- `task-hpa-manifest`를 `/plan → /work → /ship --execute`로 무리 없이 끝낼 수 있다
- 사용자가 "이제 이 흐름이 실제로 편하다"고 느낀다
- 다음 개선 항목이 문서로 분리돼 있고, v0 문서와 섞이지 않는다
